## v1.12.14-r14-hotfix1

- **安装逻辑修复**：重构 `customize.sh` 安装顺序，先执行热停机再部署文件，避免更新时服务冲突。
- **进程管理增强**：引入精准 pkill 逻辑，直接匹配二进制和服务脚本路径，确保旧进程彻底清理。
- **稳定性提升**：使用 nohup 守护热启动进程，防止安装脚本退出后服务被意外终止。
- **用户体验优化**：更新安装提示信息，更清晰地引导用户完成配置和重启步骤。

## v1.12.14-r14

- **架构级革新**：引入“虚拟指针+物理空间”分离架构。通过在 `system/bin` 建立指向工作空间的符号链接，实现不停机、不重启的逻辑更新。
- **sbc update 降临**：为控制脚本引入云端同步功能。支持直接从模板仓库抓取最新的 `config.template.json`，实现配置逻辑的 OTA 增量更新。
- **物理路径对齐**：仓库源码、模块 ZIP 与 Workspace 实现 1:1 绝对物理对齐，消灭所有相对路径陷阱。
- **用户引导增强**：完善首次安装的 `.env` 凭证初始化告知义务。
- **⚠️ 重要提示**：由于涉及系统路径软链接的物理变动，**本次更新安装后必须重启手机一次**。后续更新将不再强制重启。

## v1.12.14-r13

- **进程管理重构**：引入父进程（PPid）穿透刺杀逻辑，彻底解决模块更新时旧进程无法杀死的顽疾。
- **稳定性增强**：为所有进程操作添加非阻塞断言（|| true），确保控制脚本在任何极端状态下均能平稳运行。
- **CI 鲁棒性修复**：解决了构建任务在处理同名二进制文件时的文件冲突（Same file error）导致的编译失败。
